# Architectural Invariants

The following invariants must remain true regardless of implementation language.

1. Reality precedes interpretation.

2. Expectations are declared, not embedded in code.

3. Evaluation is deterministic.

4. Applications define meaning.

5. The Kernel authorizes progression only through evaluation.

6. The Kernel never contains application-specific behavior.

7. Observed Reality is immutable during evaluation.

8. Expected Reality is immutable during evaluation.

9. Every evaluation produces a structured result.

10. All future implementations must preserve these invariants.
