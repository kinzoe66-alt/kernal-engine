# Kernel Constitution

Version: 0.1

The Kernel is a domain-independent deterministic evaluator.

It exists to determine whether observed reality satisfies declared expectations.

The Kernel contains no business logic.

The Kernel contains no domain knowledge.

The Kernel never interprets application meaning.

Applications define meaning.

The Kernel only performs deterministic evaluation.

The Kernel produces an explicit evaluation result.

Progression requires satisfied expectations.

Observed Reality and Expected Reality are external to the Kernel.

The Kernel is stateless between evaluations.

Every evaluation must produce the same result for identical inputs.

The Kernel may evolve internally, but these principles are considered architectural invariants.
